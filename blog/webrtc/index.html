WebRTC

April 30, 2023

WebRTC

  • 웹 브라우저 간에 플러그인의 도움 없이 서로 통신할 수 있도록 설계된 API.
  • 음성 통화, 영상 통화, P2P 파일 공유 등으로 활용될 수 있다.
  • 시그널링 과정 -> P2P 연결 -> 데이타 통신 과정으로 수행된다.
  • 시그널링 과정에 TURN, STRUN 서버가 필요하다.
  • Mesh, SFU, MCU 방식의 통신 구조를 사용한다.
  • SFU, MUC 방식은 미디어 서버가 필요하다.

시그널링

  • WebRTC API 를 이용하여 통신을 하기 위해서는 SDP, ICE Candidate 정보가 필요하다. 해당 정보를 구하는 과정이 시그널링 과정이다.
  • 시그널링 서버는 직접 구현해야하며, Socket 통신으로 구현하는 것을 추천한다.
  • Mesh, SFU, MCU 의 통신 구조나, 1:1, 1:N, N:M 방식의 서비스 구조에 따른 서버 코드를 작성하면 된다.
  • 추 후 클라이언트 통제는, 시그널링 서버로 연결된 소켓 통신을 이용하건, 시그널링 서버에 관리 처리한 미디어 서버의 endpoint 정보를 이용하여 통제한다.

P2P 연결

  • 시그널링 과정에서 구해진 정보를 이용하여 피어간 P2P 연결을 한다.
  • 이 후 통신은 Point to Point 연결된 peer 끼리 통신을 주고 받는다. ( 데이타 통신에 시그널링 서버는 관연하지 않는다는 것 )
